# Content Calendar

Default launch language: English.

Default campaign: `midi_first_users`.

## Weekly Rhythm

| Day | Automation Task | Channel | Output |
| --- | --- | --- | --- |
| Monday | Prepare weekly launch packet | Website/GitHub/outbox | one blog draft, one GitHub update, one short video script |
| Tuesday | Publish or queue educational content | Blog/RSS | guide or changelog |
| Wednesday | Prepare demo content | YouTube/TikTok/Instagram outbox | short video script and shot list |
| Thursday | Publish or queue GitHub update | GitHub | release note, issue prompt, or progress update |
| Friday | Prepare feedback request | Owned social/server outbox | feedback-focused post |
| Saturday | Review metrics | results log | traffic, stars, issues, feedback |
| Sunday | Plan next week | calendar | next experiments and stopped ideas |

## First Four Weeks

| Week | Goal | Content |
| --- | --- | --- |
| 1 | Explain the project | Open Free Tools intro, MIDI Piano Trainer pitch, Audio-to-MIDI beta limits |
| 2 | Show the workflow | audio to MIDI to falling notes, upload MIDI and practice, clean vs noisy audio |
| 3 | Ask for feedback | bad conversion issue, musician feedback request, GitHub release note |
| 4 | Repeat what worked | update best-performing guide/demo, publish weekly progress, prepare external launch research |

## UTM Defaults

Use these defaults unless a channel needs a specific value:

- `utm_medium=owned`
- `utm_campaign=midi_first_users`
- `utm_content=short_demo`, `guide`, `release`, `changelog`, or `feedback`

Channel sources:

- `utm_source=blog`
- `utm_source=github`
- `utm_source=youtube`
- `utm_source=tiktok`
- `utm_source=instagram`
- `utm_source=discord`
- `utm_source=rss`

## Stop Rules

Pause or reduce a channel if:

- users complain about spam;
- platform warnings appear;
- content is duplicated too often;
- traffic quality is poor and no feedback arrives;
- the tool has a bug that blocks first-time users.
