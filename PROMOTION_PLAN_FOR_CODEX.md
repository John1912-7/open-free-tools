# Promotion Plan for Codex

Before doing promotion work, read this file and keep promotion aligned with:

1. free/open-source tools;
2. owned-channel-first automation;
3. separate Open Free Tools project accounts where possible;
4. no spam, no fake engagement, no unsolicited DMs, no browser hacks;
5. useful content, honest limits, and feedback-first positioning.

## Mission

Use automation to bring first real users to Open Free Tools without waiting for Google search traffic.

The first promoted product is MIDI Piano Trainer with the Audio-to-MIDI beta. The first launch language is English. The first audience is musicians, piano learners, MIDI users, and music-production creators.

## Channel Policy

V1 uses owned channels only:

- Open Free Tools website and blog.
- GitHub repositories, releases, issues, and project updates.
- Open Free Tools YouTube/TikTok/Instagram project accounts, only through official upload or scheduling paths.
- Open Free Tools Discord server, only through an owned server webhook or bot.
- RSS/changelog pages on the website.
- Future owned newsletter if the project needs email updates.

Do not auto-post into external Reddit communities, other people's Discord servers, Hacker News, Product Hunt, or AlternativeTo in V1. Those can be researched and prepared, but their actual launch path must be planned separately.

## Automation Rules

Allowed automation:

- Generate daily promotion packets.
- Generate weekly metric summaries and next-step recommendations.
- Draft blog posts, changelogs, video scripts, captions, and GitHub release notes.
- Update the content calendar, outbox, and results log.
- Publish only to channels with official project access and clear permission.

Not allowed:

- No unsolicited direct messages.
- No mass posting identical copy.
- No fake likes, votes, comments, follows, or coordinated engagement.
- No bypassing captcha, anti-bot systems, login gates, or platform limits.
- No browser automation that imitates a human account where the platform forbids automated account actions.
- No claims that Audio-to-MIDI quality is perfect or comparable to paid services before benchmarks support it.

## Project Accounts

Default account policy:

- Use separate Open Free Tools project accounts first.
- Keep personal accounts as fallback only if project accounts are blocked by platform setup limits.
- Store credentials outside the repository.
- Record account status in `marketing/owned-channels.md`.

## Core Positioning

Primary pitch:

> Free open-source browser tools for musicians, starting with a MIDI piano trainer and an experimental audio-to-MIDI workflow.

Honest quality line:

> Audio-to-MIDI is a beta. Clean short piano recordings work best, and bad/noisy recordings may produce wrong notes.

Main calls to action:

- Try the tool.
- Star the project on GitHub.
- Report a bad conversion.
- Share feedback.
- Support the project if it helps.

## Daily Promotion Packet

Every daily packet should include:

- one owned channel target;
- one post or content draft;
- one target URL with UTM parameters;
- one asset checklist;
- one safety note;
- publish status: `ready`, `published`, `blocked`, or `manual_or_later`;
- a short reason when blocked.

## Weekly Review

Every weekly review should summarize:

- published items;
- visits by channel;
- GitHub stars or issues;
- bad conversion feedback;
- tool clicks;
- what content should be repeated, changed, or stopped;
- next week plan.
